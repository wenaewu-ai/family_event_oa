# Changelog

All notable changes to this project will be documented in this file.

## [0.0.3.0] - 2026-04-05

### Added
- 活動詳情卡片新增「📋 查看提交明細」按鈕，所有成員（非管理員專屬）均可查看本次活動所有家庭的費用提交紀錄
- 新增 `event_all_expenses_card` 卡片，顯示所有家庭費用明細（自動過濾「加入分攤」零元佔位紀錄）

## [0.0.2.0] - 2026-04-05

### Added
- 活動卡片列表現在顯示即時計算的總費用與每戶分攤金額（原本固定顯示 $0）
- 活動詳情卡片新增「差額」欄位，每戶顯示已付與應付的差額（+ 代表可退、- 代表應補）
- 新增「🙋 加入分攤」按鈕：活動進行中、且家庭尚未提報費用時顯示，讓未預支費用的家庭也能參與分攤
- 活動已結束後自動隱藏「提交費用」與「加入分攤」按鈕

### Changed
- 結清操作現在僅限活動關帳（狀態為「已結束」）後才能執行，進行中的活動無法結清

### Fixed
- 結清狀態判斷改為白名單邏輯（必須為「已結束」才放行），防止狀態值異常時誤允許結清

## [0.0.1.0] - 2026-04-05

### Fixed
- 公積金輸入：「取消」指令現在在格式驗證前先被攔截，避免使用者輸入「取消」卻收到格式錯誤訊息
- 公積金輸入：新增金額必須大於 0 的驗證，防止輸入 0 或負數
- 結清流程：當 settle 指令格式不正確時，回覆明確的錯誤提示而非靜默失敗
- 試算表標頭解析：去除欄位名稱中的 `\n(...)` 說明後綴，與 `update_event_status` 保持一致，修正 `mark_family_settled` 無法找到正確欄位的問題
- 全形數字輸入：將 `isdigit()` 改為 ASCII 數字正則比對，防止全形數字（如 `１２３`）導致 `int()` 崩潰
